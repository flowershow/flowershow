# ADR 0006: Use `+` to encode spaces in URL slugs

**Status**: Accepted

## Context

Vault file paths often contain spaces (e.g. `My Notes/Cool Article.md`). These spaces must be encoded in URL slugs. The two candidates are:

- `%20` — the RFC 3986-standard percent-encoding for spaces in URL paths
- `+` — conventional only in query strings (`application/x-www-form-urlencoded`), but used by Obsidian Publish for path segments

## Decision

Use `+` to encode spaces in URL slugs (e.g. `/My+Notes/Cool+Article`).

The middleware normalises both `+` and `%20` to `%20` before routing. This handles two space-encoding sources:

1. **Canonical `+` URLs** — generated by the app (links in rendered pages, sidebar navigation).
2. **CommonMark `%20` URLs** — when a vault file contains a CommonMark link to a file with spaces (e.g. `[Post](My%20Notes/Cool%20Article.md)`), the CommonMark spec requires percent-encoding. Browsers follow that link with `%20` in the URL. Flowershow must serve the correct page regardless.

With everything normalised to `%20`, Next.js passes `%20` through unchanged in catch-all `params`, and the page component can reliably convert `%20` → `+` to reconstruct the `blob.appPath` DB key. A client-side `UrlNormalizer` component then replaces any remaining `%20` in the browser address bar with `+` to restore the canonical readable form.

Note: `%20` inside link *values* (before the link reaches the browser) is handled separately at render time by the `commonMarkSpaceEncoded` flag in `resolveContentLink`, which decodes `%20` back to a space before resolving the link to a slug.

## Consequences

- URLs are more readable (shorter, less visual noise than `%20`).
- Consistent with Obsidian Publish, so users migrating from Obsidian Publish get familiar-looking URLs.
- Requires the middleware↔page round-trip conversion. Any new routing code that reads `appPath` or constructs slugs must be aware of this convention.
- `+` in a path segment is technically non-standard; some strict URL parsers may not treat it as a space. This is accepted as a known trade-off.
